# Настройка Яндекс Телемост

## Проблема с мок-ссылками

По умолчанию приложение использует мок-ссылки для Яндекс Телемост, которые не работают в реальном сервисе. Это происходит потому, что для создания реальных встреч нужен API ключ.

## Решение 1: Использовать реальный API ключ (рекомендуется)

1. Зарегистрируйтесь в [Яндекс Облаке](https://cloud.yandex.ru/)
2. Создайте сервисный аккаунт
3. Получите API ключ для Яндекс Телемост
4. Добавьте ключ в файл `.env`:
   ```
   TELEMOST_API_KEY=ваш-api-ключ-здесь
   ```
5. Перезапустите сервер

## Решение 2: Использовать текущий подход (для тестирования)

Текущая версия генерирует уникальные ссылки, которые должны создавать встречи при первом открытии. Однако это может не работать стабильно.

## Решение 3: Создавать встречи вручную

Вместо автоматического создания через API, можно:
1. Создать встречу в Яндекс Телемост вручную
2. Скопировать ссылку на встречу
3. Использовать эту ссылку в приложении

## Проверка работы

После настройки API ключа:
1. Создайте комнату в приложении
2. Нажмите "Присоединиться"
3. Должна открыться рабочая встреча в Яндекс Телемост

Если встречи не создаются, проверьте:
- Правильность API ключа в `.env`
- Логи сервера на наличие ошибок
- Документацию Яндекс Телемост API
